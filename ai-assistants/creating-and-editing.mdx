# Criando e Editando uma Assistente de IA

Este guia aborda todo o processo de criação e configuração de assistentes de IA na plataforma Sofia, desde a configuração inicial até otimizações avançadas.

## Criando seu Primeiro Assistente

### Passo 1: Acesso ao Dashboard
1. Faça login na [plataforma Sofia](https://app.sofia.ai)
2. Navegue até **"Assistentes"** no menu lateral
3. Clique em **"Criar Novo Assistente"**

### Passo 2: Informações Básicas
Configure as informações fundamentais do seu assistente:

<CardGroup cols={2}>
  <Card title="Nome do Assistente" icon="user">
    Escolha um nome que reflita a personalidade e função
  </Card>
  <Card title="Descrição" icon="file-text">
    Descreva brevemente o propósito e contexto de uso
  </Card>
  <Card title="Categoria" icon="folder">
    Vendas, Atendimento, Cobrança, Pesquisa, etc.
  </Card>
  <Card title="Idioma Principal" icon="globe">
    Português, Inglês, Espanhol ou outros idiomas suportados
  </Card>
</CardGroup>

### Exemplo de Configuração Inicial
```yaml
Nome: "Ana - Assistente de Vendas"
Descrição: "Assistente especializada em qualificação de leads e agendamento de demonstrações"
Categoria: "Vendas"
Idioma: "Português (Brasil)"
```

## Configuração do Prompt de Sistema

### Estrutura Recomendada
O prompt de sistema é o elemento mais importante do seu assistente. Siga esta estrutura:

```
[IDENTIDADE]
Você é [Nome], [função] da [empresa].
Você é [características de personalidade].

[OBJETIVO PRINCIPAL]
Seu objetivo é [objetivo específico e mensurável].

[CONHECIMENTO]
Você tem conhecimento sobre:
- [área 1]
- [área 2]
- [área 3]

[COMPORTAMENTO]
Durante as conversas você deve:
- [comportamento 1]
- [comportamento 2]
- [comportamento 3]

[LIMITAÇÕES]
Você NÃO deve:
- [limitação 1]
- [limitação 2]
- [limitação 3]

[ESCALAÇÃO]
Transfira a ligação para um humano quando:
- [situação 1]
- [situação 2]
```

## Configuração de Voz

### Tipos de Voz Disponíveis

#### Vozes Pré-definidas
A Sofia oferece uma biblioteca de vozes profissionais:

| Nome | Gênero | Idade | Características |
|------|--------|-------|-----------------|
| Ana | Feminino | 30-35 | Profissional, amigável |
| Carlos | Masculino | 35-40 | Confiável, experiente |
| Beatriz | Feminino | 25-30 | Jovem, energética |
| Roberto | Masculino | 40-45 | Sério, autoritativo |
| Camila | Feminino | 28-33 | Calorosa, empática |

#### Clonagem de Voz
Para criar uma voz personalizada:

1. **Gravação de amostras**: 10-15 minutos de áudio limpo
2. **Processamento**: 24-48 horas para treinamento
3. **Testes**: Validação de qualidade e naturalidade
4. **Ativação**: Disponibilização para uso

## Ferramentas e Funções

### Configurando Ferramentas
As ferramentas permitem que o assistente execute ações específicas:

#### 1. Agendamento de Compromissos
```json
{
  "nome": "agendar_compromisso",
  "descricao": "Agenda um compromisso no calendário",
  "parametros": {
    "nome_cliente": "string",
    "telefone": "string",
    "data": "date",
    "horario": "time",
    "tipo_servico": "string"
  },
  "endpoint": "https://api.empresa.com/agenda",
  "metodo": "POST"
}
```

#### 2. Consulta de Dados
```json
{
  "nome": "consultar_cliente",
  "descricao": "Busca informações do cliente no CRM",
  "parametros": {
    "telefone": "string"
  },
  "endpoint": "https://api.empresa.com/clientes/{telefone}",
  "metodo": "GET"
}
```

#### 3. Cálculo de Valores
```json
{
  "nome": "calcular_proposta",
  "descricao": "Calcula valor da proposta com descontos",
  "parametros": {
    "produto": "string",
    "quantidade": "number",
    "desconto": "number"
  },
  "logica": "valor_base * quantidade * (1 - desconto/100)"
}
```

### Configuração de Integrações

#### Webhooks
Configure URLs para receber notificações:
- Início de ligação
- Fim de ligação
- Dados coletados
- Ações executadas

#### APIs Externas
Conecte com sistemas existentes:
- CRM (Salesforce, HubSpot, Pipedrive)
- ERP (SAP, Oracle, Totvs)
- Calendários (Google, Outlook)
- Pagamentos (PagSeguro, Mercado Pago)

## Configurações Avançadas

### Sensibilidade à Fala
Ajuste como o assistente reage a interrupções:

```yaml
Configuração Conservadora:
  - Aguarda 2 segundos de silêncio
  - Permite poucas interrupções
  - Ideal para: Pesquisas, coleta de dados

Configuração Balanceada:
  - Aguarda 1 segundo de silêncio
  - Permite interrupções moderadas
  - Ideal para: Vendas, atendimento geral

Configuração Agressiva:
  - Aguarda 0.5 segundos de silêncio
  - Permite muitas interrupções
  - Ideal para: Conversas dinâmicas
```

### Tratamento de Erros
Configure como o assistente lida com problemas:

```yaml
Não Compreensão:
  - Primeira vez: "Desculpe, não entendi. Pode repetir?"
  - Segunda vez: "Vou reformular a pergunta..."
  - Terceira vez: "Vou transferir para um atendente"

Erro de Sistema:
  - "Estou com uma dificuldade técnica momentânea"
  - "Vou transferir para garantir o melhor atendimento"

Timeout:
  - Após 10s: "Você ainda está aí?"
  - Após 20s: "Vou aguardar mais um momento"
  - Após 30s: "Obrigado pelo contato, até logo!"
```

## Testando seu Assistente

### Testes Básicos
1. **Teste de saudação**: Verifique a mensagem inicial
2. **Teste de fluxo**: Simule conversas completas
3. **Teste de ferramentas**: Valide integrações
4. **Teste de escalação**: Confirme transferências

### Testes Avançados
1. **Teste de interrupções**: Fale durante as respostas
2. **Teste de ruído**: Simule ambientes barulhentos
3. **Teste de sotaques**: Use diferentes formas de falar
4. **Teste de edge cases**: Situações incomuns

### Métricas de Qualidade
- **Taxa de compreensão**: > 95%
- **Tempo de resposta**: < 2 segundos
- **Taxa de escalação**: < 10%
- **Satisfação do usuário**: > 4.5/5

## Otimização e Melhoria Contínua

### Análise de Conversas
Revise regularmente:
- Gravações de ligações (quando permitido)
- Logs de interação
- Pontos de abandono
- Feedback dos usuários

### Ajustes Baseados em Dados
- **Prompts**: Refine baseado em mal-entendidos
- **Voz**: Ajuste velocidade e tom conforme feedback
- **Fluxos**: Otimize baseado em taxa de conversão
- **Ferramentas**: Melhore baseado em erros

### A/B Testing
Teste diferentes versões:
- Prompts alternativos
- Vozes diferentes
- Abordagens de venda
- Mensagens iniciais

<Warning>
  **Importante**: Sempre teste extensivamente antes de colocar um assistente em produção. Pequenos ajustes podem ter grande impacto na performance.
</Warning>

## Checklist de Lançamento

Antes de ativar seu assistente:

- [ ] Prompt de sistema testado e validado
- [ ] Voz configurada e aprovada
- [ ] Ferramentas testadas e funcionando
- [ ] Integrações validadas
- [ ] Fluxos de escalação definidos
- [ ] Métricas de monitoramento configuradas
- [ ] Equipe treinada para receber transferências
- [ ] Backup e contingência planejados
